SQL Server on Azure VM

Infrastructure as a Service (IaaS)

What It Is

  • Full SQL Server instance running on a Windows or Linux VM in Azure
  • You manage the OS, patching, backups, HA, and the SQL Server engine
  • 100% feature parity with on-premises SQL Server

Best For

Lift & Shift OS Access Legacy Apps SSIS / SSRS / SSAS
  • Migrating on-prem SQL Server with zero code changes
  • Workloads requiring OS-level access or custom agents
  • Apps using SQL Server features not yet in PaaS (CLR, linked servers to non-SQL sources, etc.)
  • Running SSIS, SSRS, or SSAS alongside the engine

Operational Profile

Your Management EffortHigh
SQL Compatibility100%
Scaling FlexibilityMedium

SQL Managed Instance

Platform as a Service (PaaS) — Instance-scoped

What It Is

  • A fully managed SQL Server instance with near-complete engine compatibility
  • Azure handles OS, patching, backups, and high availability
  • Deployed into your VNet for network isolation

Best For

Modernize Multi-DB Cross-DB Queries SQL Agent
  • Modernizing on-prem with minimal code changes but PaaS benefits
  • Apps that need instance-scoped features: SQL Agent jobs, cross-database queries, Service Broker
  • Organizations consolidating many databases onto one managed instance
  • Workloads requiring VNet integration & private endpoints

Operational Profile

Your Management EffortLow–Medium
SQL Compatibility~99%
Scaling FlexibilityMedium

Azure SQL Database

Platform as a Service (PaaS) — Database-scoped

What It Is

  • A fully managed, single database service built on the latest stable SQL Server engine
  • Automated patching, backups, tuning, and geo-replication
  • Multiple purchasing models including serverless with auto-pause for cost efficiency

Best For

Cloud-Native SaaS Microservices Serverless
  • New cloud-native applications designed for the cloud from day one
  • SaaS products needing per-tenant database isolation
  • Microservice architectures with one database per service
  • Dev/test workloads benefiting from serverless auto-pause

Operational Profile

Your Management EffortMinimal
SQL Compatibility~95%
Scaling FlexibilityHigh

Real-World Scenarios

See which option maps to common migration and greenfield situations.

Azure VM

Legacy ERP Lift & Shift

Your ERP vendor requires specific SQL Server builds, CLR assemblies, and Windows-scheduled tasks. Move the VM image to Azure with near-zero rework.

Azure VM

SSIS + SSRS Co-Located

ETL packages (SSIS) and paginated reports (SSRS) run on the same server. Azure VM lets you replicate this topology in the cloud.

Managed Instance

Consolidating Multi-Database Apps

Your app spans 15 databases with cross-DB joins and SQL Agent jobs. Managed Instance hosts all of them under one roof with PaaS management.

Managed Instance

Regulated Industry Migration

A financial services firm needs VNet isolation, no public endpoints, and near-100% SQL compatibility. MI checks every box with minimal re-architecture.

SQL Database

Multi-Tenant SaaS Product

You're building a SaaS app with one database per customer. Serverless auto-pauses idle trial databases, keeping costs low while scaling on demand.

SQL Database

Microservice Data Store

Each microservice owns its database. Azure SQL Database gives each service an independent, auto-tuned database with built-in geo-replication.

Cost Model Snapshot

What you're paying for in each model — actual pricing varies by region, tier, and reservations.

SQL Server on VM

  • VM compute (by size & series)
  • SQL Server license (or Azure Hybrid Benefit)
  • Managed disks (Premium SSD, Ultra Disk)
  • OS license (Windows Server)
  • Backup storage (you manage)

Managed Instance

  • vCores + memory tier (GP or BC)
  • Storage (per GB provisioned)
  • SQL license included (or AHB discount)
  • Backup storage (included up to 100% of instance size)
  • No OS/infra costs

SQL Database

  • DTU-based or vCore-based tiers
  • Serverless: pay per second of compute
  • Provisioned or serverless compute options
  • Backup storage (included up to 100%)
  • Hyperscale: scale storage independently
